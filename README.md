# ai_mind_test1
openai + json states of mind 

Эксперимент №1 по созданию симуляции мышления. Предположительно, если мы познаем состояния человеческого мышления - его можно симулировать.

![разрядка на пустоты](https://github.com/user-attachments/assets/618b772a-04c3-4e73-9214-a281de76d6dc)

Описанные состояния мы смакуем регулярно, они сопровождают нас в течение всей жизни. Однако при попытке симулировать этот цикл мы сталкиваемся с проблемой реализации. 
Допустим, что на вход подаётся начальная мысль, которую мы сгенерируем с помощью случайных слов и БЯМ. Далее активизиурется само мышление, и происходит категоризация данной мысли, по древу можно отфильтровать состояние, найти связанные, забить это в цикл, в котором каждую итерацию (условно с задержкой на 10 секунд) - происходит путешествие по древу. Ну например: Реплика "Ты опять ошибся" -> активирует Уточнение (что имеется в виду?), Тревога (что будет?), Сожаление (жаль, что так вышло) -> далее активируются О былом, Гнев, Сравнение и т.д. Условимся, что будет максимум 3 итерации переключения состояний. 
Следующая проблема реализации - взаимодействие мыслительных состояний бота с входящим контекстом, предположительно должно быть несколько асинхронных циклов... Например можно разбить проблему на слои: эмоциональный цикл, обработка стимулов, механизм принятия решений. Можно предложить архитектуру с тремя параллельными процессами: оценка входящих событий, применение эффектов состояний, генерация реакции. Отсюда вытекает ещё одна проблема - взаимодействия этих циклов. Предположительно у людей на этом моменте есть некое внутреннее внимание(осознание), которое позволяет переключаться между различными мыслями в контексте (в кратковременной памяти), мы берём различные мысли и соотносим с окружающим миром. 
Также у нас имеется модель окружающего мира, который мы постоянно пытаемся Уточнить (дополнить) деталями с помощью органов чувств - но в случае с чат-ботом, анализирующем текст, единственное осязание окружающего мира - это вопрос-ответное мышление, с помощью которого бот мог бы отвечать себе на вопросы, формировать базу данных о мире, окружающим его (ну или создавать в отдельной папке json файл с заметками). Отсюда ещё одна проблема - согласование различных элементов из базы данных соединяться друг с другом, формируя ассоциации, которые пробуждают из долговременной памяти тот или иной образ, который формирует ту или иную тему для размышлений.
При этом бот отвечает не на каждый запрос, а при решении ответить (Мы сначала обдумываем нужно ли нам отвечать, думаем прежде чем сказать), когда совокупная активность определенных узлов превышает порог.

Можем ли мы с помощью древа мыслительных состояний и запросов к БЯМ - создать симуляцию мышления? Цель эксперимента выяснить это.
